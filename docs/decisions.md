## K. 経費・交通費（次フェーズ）

### K1. ユーザー区分別の使える申請種別

| ユーザー区分 | 通勤交通費 | 事前申請 | 経費申請 |
|---|:---:|:---:|:---:|
| **SES 事業部** | ✅ | ❌ | ❌ |
| **管理部（一般ロール）** | ✅ | ✅ | ✅（事前申請とペア必須） |
| **管理部（admin ロール）** | ✅ | ❌ | ✅（ペア不要） |

- 一般ロール（管理部）の経費申請は、必ず承認済みの事前申請とペアで登録
- admin は事前申請スキップで直接経費申請可能
- 上限額は **設けない**
- **作り込みは次フェーズ**（事前申請⇔経費申請のペア UI、ワークフロー詳細は別途設計）

### K2. 申請可能期間

| 申請種別 | 申請可能期間 |
|---|---|
| 通勤交通費（都度） | 利用日が申請日から **過去 3 ヶ月以内**、未来日不可 |
| 通勤交通費（1 ヶ月定期） | 開始日が **申請日の前月 1 日〜翌月末日** |
| 通勤交通費（3 ヶ月定期） | 開始日が **申請日の前月 1 日〜翌月末日** |
| 経費申請（事後精算） | 発生日から **過去 3 ヶ月以内**、未来日不可 |
| 事前申請 | 発生予定日が **未来日のみ**、過去日不可 |

- 期間外の申請はバリデーションエラーで弾く
- 遅延は管理者が代理登録で対応

### K3. 金額バリデーション

- 負数は弾く（0 円以下は不可）

## 🚩 交通費申請システム大改修（本セッションで設計、優先度: 最高）

### 背景

現状の `ExpenseRequest` / `ExpenseItem` は「都度申請のみ」対応で、定期券の種別・有効期間・通勤スタイル管理は一切無い。K2・K3 の要件を満たすには大幅改修が必要。

### データモデル変更

#### 1. `Employee` テーブルにフィールド追加

```prisma
model Employee {
  // ... 既存フィールド ...
  commuteStyle String? @map("commute_style") @db.VarChar(20)
  // "onetime" | "monthly" | "three_month" | null
}
```

#### 2. `ExpenseItem` テーブルにフィールド追加

```prisma
model ExpenseItem {
  // ... 既存フィールド ...
  kind        String    @default("onetime") @db.VarChar(20)
  // "onetime" | "monthly_pass" | "three_month_pass"
  passEndDate DateTime? @map("pass_end_date") @db.Date
  // 定期の場合のみ、定期券の終了日
}
```

- `expenseDate`: 都度なら利用日、定期なら開始日として兼用
- 都度と定期を同じ `ExpenseRequest` に混在させて申請可能

### 申請フォーム（`/mypage/expense`）再設計

- 既存フォームは都度明細のみ → 定期明細セクションを追加
- `[＋ 定期代を追加]` ボタン: 種別（1 ヶ月 / 3 ヶ月）、開始日、区間、金額 を入力
- `[＋ 都度代を追加]` ボタン: 既存の都度明細（日付、区間、金額）
- 合計金額は自動計算、確認モーダル → 送信
- **API 連携を実装**（現状モック）

### バリデーション（`expense.service.ts`）

- K2 の期間制約を申請種別ごとに実装
- K3 の負数バリデーション
- 3 ヶ月定期重複申請エラー（同社員の有効期間が重なる定期申請を弾く）
- 1 ヶ月定期も同様に重複エラー

### アラート仕様の書き換え（`mypage/page.tsx` + `attendance.service.ts`）

既存の `expenseMissing` フラグを **通勤スタイル別** に分岐:

#### 都度ユーザー (`commuteStyle = "onetime"`)

- アラートなし（過去 3 ヶ月遡れるので忘れても救済可能）

#### 1 ヶ月定期ユーザー (`commuteStyle = "monthly"`)

- 毎月 15 日時点で判定 → 当月の定期 or 都度申請が 1 件もなければアラート
- 月末日に再アラート

#### 3 ヶ月定期ユーザー (`commuteStyle = "three_month"`)

- 「現在月をカバーする定期申請（有効期間に現在月を含む）or 当月の都度申請」が 1 件もない場合にアラート
- 先行申請（例: 1 月に 1/1〜3/31 の定期を申請 → さらに 3 月中に 4/1〜6/30 の定期を先行申請）もカバー
- 月 15 日・月末日でアラート判定

アラート文言: 「定期券の申請はお済みですか？」
クリックで `/mypage/expense` に遷移

### 社員詳細画面での `commuteStyle` 設定

- `/admin/employees/[id]` の「契約・給与」タブに追加
- セレクタ: 都度 / 1 ヶ月定期 / 3 ヶ月定期 / 未設定

## 検証方法

### 交通費申請（優先度 S）

1. Prisma マイグレーション実行: `npx prisma migrate dev`
2. 社員マスタで `commuteStyle` を設定（都度 / 1 ヶ月 / 3 ヶ月 で 3 人分テスト）
3. `/mypage/expense` で定期代 + 都度代を混在させて申請 → 承認 → 正しく保存
4. 期間外申請（3 ヶ月先の定期など）がエラーで弾かれる
5. 負数がエラーで弾かれる
6. 1 ヶ月定期ユーザーで、15 日を過ぎた時点で申請無し → mypage にアラート表示
7. 3 ヶ月定期ユーザーで、先行申請（4/1〜6/30 を 3 月中に申請）→ 4 月になってもアラート出ない
8. 日付を月末にして再アラート確認

### J2/J4（優先度 A）

1. 社員詳細画面で所定労働時間・固定残業・還元率を設定
2. 給与確定 → 計算結果がハードコードから外れて社員別設定を反映している

### L1/L2（優先度 A）

1. 社員追加時に付与方式を選択 → 有給残数が自動付与される
2. 前職引継ぎを選んだ場合、入力した残日数がそのまま設定される

### 全体

- `npx tsc --noEmit` でビルドエラーなし
- 主要画面を Preview MCP で目視確認
